TASK MANAGEMENT APP

BY TIYA WANG

Introduction

When facing heavy school tasks, most college students have trouble how properly planning out their time. The failure to manage time might result in stress, low-quality work, waste of time, or bad grades. In order to better achieve their goals in a shorter period of time, I designed a task management mobile app for college students.

Ideate

App Features

I began by brainstorming the app features with my classmates. In a group of three, we generated some basic functionalities: deadlines, notifications, and categories. Combining each group‚Äôs ideas gave me a general blueprint for the app.



To avoid lengthy and unnecessary work, I identified the MVPs my app should have with Professor's help:

  • Categories (Different Courses/Different Tasks)
  • Priority (Sorted by Your Urgency)
  • Reminders (Notification/Calendar)
  • Status (Progress/Completion)
  • Deadlines (Due Dates)
  • Customization (Ability to Add Tasks)

Mind Map

Irene, Cindy, and I drew a mind map to visualize all the features the app will have. We also drew connections between each feature putting them into a broad category.

Design

My Aims:

  • Neat looking
  • Straightforward icons
  • Contains all MVPs
Wireframes

Based on the MVPs, I designed a wireframe with four main tabs in the menu: home, calendar, setting, and self.



User Flow

Since my wireframes already have the main functions, I just added a few more pages to make it look more like an app. For example, I created a login page, and also added more details to the original features to enrich the content. Finally, I added interactions between the icons and pages. This made it so much easier for the later prototyping. I could easily follow the app outline and needed only add some colors and design the style.

Reflection

User Feedback

There is one problem using my app: some of the user flows are not connected to each other. The solution is to consider all the interactions and add all the user flows to the connected pages and buttons.

What did I learn?

I learned the whole process of solving a problem from ideation to the final product. From this project, I learned how to draw wireframes and user flow. Most importantly, I got to learn to build a mockup app using Figma.

What will I do in the future?

In the future, I would like to make more components and variants instead of making tons of frames. I also want to make most of the app elements interactive, such as the input box, where you can type the course or task name. If I have more time, I want to add a few more pages such as a login page. For future projects, I will save my progress more often for easier revision.
